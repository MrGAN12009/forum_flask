# Руководство по участию в проекте

Спасибо за интерес к улучшению проекта форума!

## Как внести вклад

### Сообщения об ошибках

Если вы нашли ошибку:
1. Проверьте, не была ли она уже сообщена
2. Создайте новый Issue с подробным описанием
3. Приложите скриншоты и логи если возможно

### Предложение новых функций

1. Создайте Issue с описанием функции
2. Объясните, зачем эта функция нужна
3. Дождитесь обсуждения перед началом разработки

### Pull Requests

1. Форкните репозиторий
2. Создайте ветку для вашей функции (`git checkout -b feature/amazing-feature`)
3. Сделайте коммиты с понятными сообщениями
4. Убедитесь, что код работает
5. Создайте Pull Request

## Стандарты кода

### Python
- Следуйте PEP 8
- Используйте понятные имена переменных
- Добавляйте docstrings к функциям
- Комментируйте сложную логику

### HTML/CSS
- Используйте Bootstrap классы где возможно
- Поддерживайте адаптивность
- Проверяйте кросс-браузерность

### JavaScript
- Используйте современный ES6+ синтаксис
- Обрабатывайте ошибки
- Добавляйте комментарии

## Структура коммитов

Используйте следующий формат:
```
тип(область): краткое описание

Подробное описание изменений (опционально)
```

Типы:
- `feat`: новая функция
- `fix`: исправление ошибки
- `docs`: изменения в документации
- `style`: форматирование кода
- `refactor`: рефакторинг кода
- `test`: добавление тестов
- `chore`: обновление зависимостей и т.д.

Примеры:
```
feat(chat): добавлена поддержка эмодзи в сообщениях
fix(auth): исправлена проверка уникальности email
docs(readme): обновлена инструкция по установке
```

## Тестирование

Перед отправкой PR убедитесь что:
- [ ] Приложение запускается без ошибок
- [ ] Новая функция работает как ожидается
- [ ] Не сломана существующая функциональность
- [ ] Проверена работа в разных браузерах (если фронтенд)

## Вопросы?

Если у вас есть вопросы, создайте Issue с тегом `question`.
